01 — Problem

Every team has the same bottleneck.

You have a repo. You have tasks. But writing boilerplate, scaffolding, and repetitive integrations chews up the best hours of your best engineers — hours that could go toward actual product differentiation.

Traditional approaches all hit walls eventually.

🤖

AI Pair Programmers

Great for single-file edits. Fall apart on multi-file changes that span architecture decisions.

📦

Scaffold Tools

Generate the skeleton. Leave the muscles and nerves for you to wire up manually.

👷

Offshore / Bounty

High coordination overhead. Quality varies wildly. Context transfer destroys momentum.

🔄

Scripts & Templates

Each one is a one-off. Maintenance burden grows faster than the value they provide.

02 — Approach

Decompose the problem. Solve it piece by piece.

CAP doesn't try to solve "write code" in one step. It breaks the workflow into a state machine where each stage is handled by a specialized component — and failures at any stage are recoverable.

1

Submit a Task as Intent, Not Spec

Users describe what they want in plain language. The platform handles routing, scheduling, and context injection. No YAML schemas, no ticket templates — just a goal and a repo.

2

Containerize the Execution Environment

Each task runs in an isolated Docker container. No cross-contamination between projects. Cold-start a fresh environment for every task. Workers are stateless — kill and restart without side effects.

3

Route to the Right Model Dynamically

Different tasks need different model capabilities. CAP's llmrouter automatically picks between MiniMax M2.7 and Zhipu GLM based on task complexity — with fallback chains so a single model outage can't block a task.

4

Commit Through a Git Container, Not a Worker

Workers write files. They don't run git. A dedicated Git Container per project handles all version control — making concurrent tasks to the same repo safe and serialized without distributed locks.

5

Expose Everything via MCP for AI-to-AI Orchestration

CAP's own API is surfaced as MCP tools. Any AI assistant (Claude Code, OpenClaw, custom agents) can submit tasks, check status, review diffs, and approve results — without leaving the tooling context.
